Maritime route estimation

Examples

  • A single route: seaws?opos=5.3,43.3&dpos=121.8,31.2. See on a map.
  • Several routes: seaws?opos=5.3,43.3,10.0,-1&dpos=121.8,31.2,120,0. See on a map.

URL parameters

Parameter Description Possible values Default value
ser Set to rou for maritime route estimation. Set to rouinfo to show some information on the service. rou, rouinfo rou
opos and dpos The origins and destinations coordinates (longitude,latitude) pairs. Comma separated. NB: The numbers of values for both parameters should be the same, and should be even.
g Set to 0 if the geometry do not need to be returned, 1 otherwise. 0,1 1
d Set to 0 if the distance do not need to be returned, 1 otherwise. 0,1 1
res The resolution of the output, in km. Small value return more detailled estimation but are slower. Larger values return more simplified estimations but faster. 5, 10, 20, 50, 100 20
suez Set to 1 to allow the route to go through the Suez channel. 0, 1 1
panama Set to 1 to allow the route to go through the Panama channel. 0, 1 1

Output format

The output is a single object if only one route if requested, or a list. For each route, the object properties are the following:

  • status: The status of the answer. Possible values are ok, empty, or error.
  • geom: The route geometry as a GeoJSON MultiLineString.
  • dist: The route length, in km.
  • message: In case of error, a message explaining it.
